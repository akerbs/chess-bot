# Инструкция по настройке Web App

## Проблема: Web App не открывается

Если вы видите текстовую доску вместо полноценного интерфейса, значит Web App не настроен.

## Решение

### Вариант 1: Локальная разработка с ngrok

1. **Установите ngrok** (если еще не установлен):
   ```bash
   # Windows (через Chocolatey)
   choco install ngrok
   
   # Или скачайте с https://ngrok.com/download
   ```

2. **Запустите Web App сервер**:
   ```bash
   cd chess-bot
   npm run webapp
   ```
   Сервер запустится на `http://localhost:3000`

3. **Запустите ngrok** (в отдельном терминале):
   ```bash
   ngrok http 3000
   ```

4. **Скопируйте HTTPS URL** от ngrok (например, `https://abc123.ngrok.io`)

5. **Добавьте в `.env`**:
   ```
   WEB_APP_URL=https://abc123.ngrok.io
   ```

6. **Перезапустите бота**:
   ```bash
   npm run dev
   ```

7. **Отправьте `/start` боту** - теперь должна появиться кнопка "♟️ Играть в шахматы"

### Вариант 2: Хостинг на сервере

1. Загрузите файлы из директории `webapp` на ваш веб-сервер
2. Убедитесь, что файлы доступны по HTTPS
3. Добавьте URL в `.env`:
   ```
   WEB_APP_URL=https://your-domain.com
   ```

### Вариант 3: Использование бесплатного хостинга

Можно использовать:
- **Netlify** - загрузите папку `webapp` через drag & drop
- **Vercel** - подключите GitHub репозиторий
- **GitHub Pages** - загрузите файлы в репозиторий

## Проверка

После настройки:

1. Отправьте `/start` боту
2. Должна появиться кнопка **"♟️ Играть в шахматы"**
3. При нажатии откроется полноценный интерфейс с drag and drop

## Если не работает

1. Проверьте, что `WEB_APP_URL` в `.env` начинается с `https://`
2. Убедитесь, что веб-сервер запущен и доступен
3. Проверьте логи бота на наличие ошибок
4. Убедитесь, что URL доступен из браузера

## Важно

- Web App требует HTTPS (не работает с HTTP)
- URL должен быть доступен из интернета (не localhost)
- Для локальной разработки используйте ngrok или аналогичный сервис
